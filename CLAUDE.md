## Constitution (Immutable Principles)

> Иммутабельные правила проекта. Конкретные технологические решения и trade-off'ы — в `docs/{epicId}/plan.md → Key Decisions`. Здесь — то, что нельзя нарушать ни на каком эпике.

### 1. Копирайт уровня зрелого редактора, не нейросети

Каждое слово на сайте должно проходить тест «написал бы это редактор архитектурного журнала?». Запрещены штампы («уют и функциональность», «создаём пространства, в которых хочется жить», «стиль и комфорт», «ваш дом мечты», «индивидуальный подход», «превращаем в произведение искусства», «не просто X, а Y», «это больше чем дизайн»), безличные обобщения, эмоциональные апелляции без конкретики, риторические перечисления через тире, выдающие LLM. Голос — личный, первое лицо единственное число, тон — сдержанный, материальный, конкретный.

_Rationale_: целевая аудитория (платёжеспособный заказчик дизайна интерьеров средний+ сегмента) умеет отличать осмысленный авторский текст от копирайтерской рутины. Слабый текст здесь не нейтрален — он активно отталкивает.

_Уточнение (ep02-design-and-copy, 2026-05-26):_ запрещён placeholder-текст («лорем ипсум», fake-английский, неотредактированный LLM-черновик) в любом коммите, попадающем в `main`. Любая страница, опубликованная без человеческого первичного драфта и человеческой редактуры, считается несовместимой с Принципом 1, даже если она формально не содержит запрещённых штампов. Промежуточные ветки и preview-деплои этому ограничению не подлежат — речь только о `main`.

### 2. Админка для нетехнического пользователя

Любой контент, который автор может менять (тексты страниц, проекты, фото, контакты, SEO-поля), должен меняться через WYSIWYG-интерфейс админки без участия разработчика и без знания Markdown/HTML/Git. CLI, конфиг-файлы, «погугли regex» — запрещены как путь обновления контента. Если для какого-то редактирования нужно вмешательство разработчика — это баг в архитектуре, не норма.

_Rationale_: Светлана — не технический человек. Если для смены телефона на странице «Контакты» нужно позвать брата, сайт мёртв через 3 месяца.

### 3. Lighthouse 95+ — не цель, а пороговый билд-чек

Любой коммит в `main`, который роняет любую из четырёх метрик Lighthouse mobile (Performance / A11y / Best Practices / SEO) ниже 95, не должен попадать на прод. Это автоматизируется через CI-check, блокирующий merge.

_Rationale_: сайт-визитка для платёжеспособной аудитории. Медленный сайт = потерянный клиент. Раз нарушим — постепенно сползём.

### 4. Контент с тобой навсегда

Источник истины контента — Markdown в нашем Git-репозитории. Никаких SaaS-CMS, где контент живёт у вендора и теряется при закрытии аккаунта. Любое решение, которое тащит контент в чужую базу, отвергается.

_Rationale_: в реалиях санкций РФ + волатильность SaaS-вендоров vendor lock-in на контент дороже любой удобной UX-админки. Markdown в Git переживёт любой кризис.

### 5. Бюджет нулевой по умолчанию

Все слои стека на старте — free tier. Любая ежемесячная подписка >$5 должна иметь обоснование в `research.md` или `plan.md` соответствующего эпика. Суммарный потолок инфры — $25/мес.

_Rationale_: накопление мелких подписок ($10 здесь, $15 там) убивает экономику персонального проекта быстрее, чем кажется. Принцип заставляет каждый раз спрашивать «это правда необходимо?».

### 6. Простота сильнее красоты архитектуры

Между двумя решениями, одинаково решающими задачу, выбираем то, у которого меньше движущихся частей, меньше зависимостей, меньше кода. Никаких микросервисов, никаких контейнерных оркестраторов, никаких базовых «на будущее». Если решение нельзя объяснить в одном абзаце — оно слишком сложное.

_Rationale_: один автор, один администратор, тонкий проект. Любая сложность — это будущая боль при передаче или при возврате в код через год.

### 7. Честность подачи

Сайт не имитирует уровень опыта, которого нет. Концепт-проект называется концепт-проектом, диплом о переподготовке называется дипломом о переподготовке, junior называет себя junior через выбор слов и подачу, не лживым словом. Никаких stock-фото «красивых интерьеров», никаких упоминаний клиентов, которых нет, никакого «we have been delivering excellence for years».

_Rationale_: платёжеспособная аудитория считывает фальшь мгновенно и теряет доверие необратимо. Честный голос junior, у которого виден вкус, продаётся лучше, чем имитация senior, который виден насквозь.
