# Project Vision

> Spine document — читается `/my-research` и `/my-plan` перед планированием новой работы. Обновляется только когда направление проекта или ядро архитектуры реально меняется, не на каждый эпик.

## Product Vision

Личный сайт-портфолио для начинающего дизайнера интерьеров **Светланы Головиной** (@svgodesign, РФ, самозанятая/НПД). Решает две задачи одновременно: показывает вкус, подход и насмотренность на уровне зрелого международного портфолио — и работает как канал привлечения частных заказчиков (средний+ сегмент собственников квартир/домов в РФ/СНГ).

Стратегическая задача — продать вкус и подход, а не объём опыта. На старте: 4 концепт-проекта (рендеры в Homestyler с отретушированным watermark) и диплом о профессиональной переподготовке (ООО "ДИСКИЛЛ", май 2026). Подача "Universal" — без явных меток concept/realized; метаданные проекта (год, тип, метраж) говорят сами за себя. Сайт компенсирует тонкий материал подачей: типографикой, концептуальным копирайтом, визуальной осознанностью, минимумом UI-шума. Услуги: полный дизайн-проект, авторский надзор, консультации — управляемы через админку.

Целевая аудитория сайта — частные заказчики РФ/СНГ, ищущие дизайнера для квартиры/дома и оценивающие вкус до первого касания. Вторичные — потенциальные партнёры (студии, архитекторы) и СМИ. Сайт работает в связке с активными каналами автора в Telegram и VK; он не приводит клиентов "с холода" в первый месяц, но обслуживает доверие тех, кто уже видел работы в соцсетях.

## Architecture Map

- **Front:** Astro 4+ — статический генератор, ~90% меньше клиентского JS, чем Next.js, Core Web Vitals из коробки. Лучший матч для портфолио без серверной логики.
- **CMS:** Decap CMS (резерв: Keystatic) — git-backed, MIT, контент в Markdown в репозитории. Vendor-нейтрально, нулевая ежемесячная плата, переживает любую миграцию.
- **Контент-коллекции в CMS:** `projects` (CRUD проектов с галереей и метаданными), `services` (CRUD списка услуг — на старте 3 карточки, расширяемо), `pages` (Hero, About, Services intro, Contact intro), `contacts` (4 канала: Telegram, TG-блог, VK, max.me — редактируемо), `seo` (мета-поля per page).
- **Хостинг:** Cloudflare Pages free tier на превью-поддомене `*.pages.dev` в ep01; собственный домен подключается в ep03. CF proxy — обвязка против РКН-блокировок IP-диапазонов конкурентов.
- **Резервный хостинг:** Timeweb Cloud Apps / Selectel — на случай массовой блокировки CF в РФ. Архитектура полностью переносима (статика + Markdown).
- **Изображения:** build-time оптимизация через Astro `<Image />` (sharp → AVIF/WebP), отдача с CF edge. Оригиналы — отретушированные от watermark Homestyler JPEG, заливаются через Decap (GitHub API, лимит ~50MB на файл). Внешний image CDN отложен до объёмов 500+ фото.
- **Формы:** Cloudflare Worker → Telegram Bot API `sendMessage` в чат владельца + email-копия через Яндекс SMTP. Никаких западных form-сервисов. Cookie/152-ФЗ banner — обязателен (сестра — оператор ПДн как самозанятая).
- **TG-виджет "Последние из блога":** статический парсинг RSS-фида или Telegram Widget на главной/в подвале, ссылка на @Golovina_design_ambersoftloft.
- **Аналитика:** Yandex.Metrica (приоритет, бесплатно, тепловые карты + вебвизор). GA4 — нет.
- **Auth админки:** GitHub OAuth через Decap auth proxy на CF Worker.
- **CI/CD:** push в `main` → CF Pages build → edge через 2–4 минуты.

## Epic Roadmap

**ep01-portfolio (этот эпик) — Каркас сайта, инфраструктура, превью.** Поднимаем Astro + Decap CMS, деплоим на бесплатный `*.pages.dev` Cloudflare Pages (без покупки домена), делаем базовую типографику и сетку, страницы-шаблоны для 4 проектов, About, Услуги (3 услуги управляются через CRUD), Контакты с 4 каналами (Telegram личный, TG-блог, VK, max.me), форма заявки через Telegram Bot, виджет "Последние из TG" в подвале. На выходе — рабочий сайт под превью-доменом, в котором сестра через визуальный редактор может править весь контент. **Доп. артефакт эпика:** `docs/guide-for-svetlana.md` — пошаговый гайд на русском (как пользоваться админкой, какие домены рассматривать, как зарегистрировать домен, как перенести сайт с превью-домена на свой). Этот эпик не включает финальный визуальный код, финальные тексты, hero-фото и собственный домен.

**ep02-design-and-copy (следующий) — Дизайн и копирайт.** Финальная типографическая система через 2–3 moodboard'а на выбор сестры (холодный скандинавский / тёплый тактильный / контрастный архитектурный); страница проекта в окончательном виде; написанные тексты Hero/About/Services собственным голосом без LLM-штампов; решение по Hero-фото (типографическая заглушка vs кадр материала vs лучший проектный кадр); один проект полностью оформлен как pilot для остальных. Здесь же финализируется этическая подача concept-проектов.

**ep03-content-and-launch (после ep02) — Наполнение, домен, запуск.** Сестра через админку самостоятельно заводит проекты с отретушированными фото и собственными текстами по шаблону; по гайду из ep01 регистрирует выбранный домен (вероятнее всего `golovinadesign.ru`); по гайду переносит сайт с `*.pages.dev` на свой домен (DNS + переадресация). SEO-проверка, проверка Lighthouse 95+ из РФ ISP, публичный запуск.

Возможные следующие эпики (далее, не часть этой первой волны): EN-локализация; журнал/блог как полноценный раздел сайта; визуальный конструктор страниц проектов; интеграция бронирования первичной консультации.

## Open Questions

Изначальный список Open Questions закрыт сессией Q&A 2026-05-25 (см. `docs/ep01-portfolio/research.md` → таблица "Решения"). Остаются вопросы, релевантные будущим эпикам, не блокирующие ep01:

1. **Эстетика и визуальный камертон (ep02):** холодный скандинавский / тёплый тактильный / контрастный архитектурный. Сестра выбирает по 2–3 moodboard'ам.
2. **Hero-фото (ep02):** типографическая заглушка vs кадр материала со стока vs лучший проектный кадр.
3. **Выбор домена (ep03):** `golovinadesign.ru` (рекомендация) / `golovina.design` / `svgodesign.ru` / другой. Решает сестра по гайду.
4. **EN-локализация (после ep03, отдельный эпик):** когда появится реальный международный поток.
5. **Полноценный блог на сайте (после ep03):** перерастёт ли TG-виджет в свой `/journal`.
6. **Объём `guide-for-svetlana.md` (ep01):** минимальный (домен + админка + перенос) или расширенный (включая ретушь watermark, гид по копирайту, по фотосессии).

## Non-goals

Явно вне границ проекта (как минимум в ep01–ep03):

- Не сайт-конструктор. Не Tilda/Wix/Squarespace — категорически. Мы делаем кастомный сайт с собственным визуальным кодом.
- Не приложение. Никаких SPA, никаких "личных кабинетов клиентов", никаких "интерактивных подборщиков стилей".
- Не SaaS контент-хаб. Никаких Sanity/Contentful/Hygraph — контент остаётся у нас, в Markdown в Git.
- Не маркетплейс / не каталог поставщиков. Это персональное портфолио одного автора.
- Не EN-локализация в MVP. Технически открыто, но контент только на русском в первой волне эпиков.
- Не e-commerce. Никаких покупок дизайн-пакетов через сайт. Все коммерческие переговоры — оффлайн через Telegram.
- Не имитация realized-портфолио. Подача "Universal" — без явных меток concept/realized, но и без лживых атрибутов "реализован, фотограф Х". Диплом о переподготовке подаётся как переподготовка; junior говорит языком junior'а с собственным взглядом, а не "we have been delivering excellence since 1995".
- Не псевдоним "Софья" — везде используется реальное имя Светлана Головина.
- Не платная инфра > $25/мес суммарно. Любой layer > $5/мес требует обоснования в research эпика.
- Не сторонняя web-аналитика помимо Yandex.Metrica в MVP (GA4, Hotjar, Plausible — позже, если будет смысл).
- Не stock-фото "красивых интерьеров" в любом виде на любой странице.
